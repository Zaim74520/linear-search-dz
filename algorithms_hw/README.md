# Домашнее задание: Алгоритмы и структуры данных
## Пункт 02. Анализ временной и пространственной сложности бинарного поиска

### Временная сложность

- **В худшем и среднем случае: O(log n)**  
  На каждой итерации алгоритм отбрасывает половину оставшихся элементов. Например, для списка из 1 024 элементов потребуется не более 10 сравнений, потому что $\log_2 1024 = 10$.

- **В лучшем случае: O(1)**  
  Если искомый элемент находится ровно в середине массива при первой проверке, поиск завершается за одну операцию.

### Пространственная сложность

- **O(1)** (константная память)  
  Алгоритм использует только три переменные для хранения индексов: `left`, `right`, `mid`. Объём дополнительной памяти не зависит от размера входного списка.

### Вывод

Бинарный поиск эффективен для больших отсортированных массивов: количество операций растёт логарифмически, а не линейно. Поэтому он значительно быстрее линейного поиска ($O(n)$) на больших данных, но требует, чтобы массив был отсортирован.
