# Линейный поиск: замеры времени

Проект для демонстрации сложности алгоритма линейного поиска O(n).

## Результаты

Проведены замеры времени выполнения поиска в худшем случае (элемент отсутствует) для разных размеров списка.

| Размер списка (n) | Время выполнения (сек) |
| :--- | :--- |
| 10 | ~0.00000430 |
| 100 | ~0.00000970 |
| 1000 | ~0.00008490 |

График зависимости времени от размера списка:

![График](docs/graph.png)

## Вывод

Наблюдается линейный рост времени выполнения при увеличении размера входных данных. Это подтверждает теоретическую оценку сложности алгоритма как **O(n)**.
