# Анализ временной и пространственной сложности линейного поиска

## Временная сложность: $O(n)$

Время работы алгоритма напрямую зависит от количества элементов $n$ в списке.

- **Худший случай ($O(n)$)**. Искомый элемент отсутствует в списке либо находится в самом конце. Алгоритму придётся выполнить $n$ сравнений (пройти по всему массиву). Именно этот случай определяет общую сложность алгоритма, так как задаёт верхнюю границу времени выполнения.
- **Средний случай ($O(n)$)**. При равномерном распределении данных элемент находится примерно в середине списка. Количество операций составит $n/2$. В нотации Big O константы отбрасываются ($\frac{1}{2}n \rightarrow n$), поэтому сложность остаётся линейной.
- **Лучший случай ($O(1)$)**. Искомый элемент находится на первой позиции. Алгоритм выполнит всего одно сравнение и завершится. Однако при оценке эффективности алгоритма ориентируются на худший и средний случаи.

**Вывод:** Основная временная сложность линейного поиска — **$O(n)$**. Это значит, что время работы растёт линейно с ростом количества данных.

## Пространственная сложность: $O(1)$

Алгоритм не требует выделения дополнительной памяти, пропорциональной размеру входных данных.

- В процессе работы используются только фиксированные переменные (счётчик цикла и текущее значение элемента).
- Не создаются копии списка, не используется рекурсия и не хранятся промежуточные результаты.
- Объём потребляемой памяти остаётся постоянным вне зависимости от размера списка ($n$).

**Вывод:** Пространственная сложность линейного поиска — **$O(1)$** (константная).
